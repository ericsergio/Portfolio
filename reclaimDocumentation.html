Class: NotebookUI

Access Modifier Method Description
private OnEnable Callback functions for buttons established

public ShowMainMenu opens the closed notebook menu while closing everything else
private ExitBtnClicked Runs the CloseMenu method on the exit button’s click
public CloseMenu closes the closedNotebookMenu page
private SettingsClicked Runs UIExtensions Display method for associated page, setting the associated page’s flex to true for that page. Also sets that page’s bottom positioning attribute to the calculated percentage value found in screenRatioPercentValue. Every time one of these buttons get clicked, Tab Navigation gets run as well so that the pages have the working tabs for the user to use to switch from page to page while the notebook is open.
private ConversationsClicked See SettingsClicked
private InventoryClicked See SettingsClicked
private TasksClicked See SettingsClicked
public TabNavigation Creates the tabs dynamically when the notebook gets opened, this method is where the callbacks are established for each tab so that the tabs are always listening via OnTabButtonClicked. The index of the tab gets passed to OnTabButtonClicked based on which tab is clicked such that OnTabButtonClicked can use the index to know which page should open and which pages should close.
private OnTabButtonClicked This method takes in the clickEvent and index number of the tab that was clicked which is how it knows which page to open and which pages to close

Class: UIController

Access Modifier Method Description
private Update Runs PreventNotebookBeforeDialogue and RestrictInputWhileUIIsOpen within the OnUpdate Method.
private PreventNotebookBeforeDialogue This method turns on or off the ability to take in input by referring to the PlayerMovement Class’s CanHandleInput method. If the game is in a state where the player shouldn’t move because the dialogue is open then set the UIExtensions.Display method to false for any page.
private RestrictInputWhileUIIsOpen This method creates percentages of the screen in which the notebook consumes and blocks raycasting depending on the coordinates of the mouse position. It checks which pages (if any) are currently open and then if they are open then depending on which is open (the main menu closed notebook page which is less wide or any of the open notebook pages) boundaries are established that fall outside of the area of the screen that the notebook takes up then ray casting should be blocked. This means that if the notebook’s positioning or size changes in the future then this will have to be adjusted as well. See class comments.

Class: UIExtensions

Access Modifier Method Description
public Display This is a simple helper class method that is used to toggle the notebook pages or main menu to be visible or not using its Flex attribute. A page can either be set to DisplayStyle.Flex or DisplayStyle.None, an enabled bool is passed to set the page’s state.

Class: SettingsController

Access Modifier Method Description
private OnEnable Binds the SettingsController to the GeometryChangedEvent in a call back such that it is listening to the geometry being changed by being opened or closed.
private OnGeometryChangeEvent For each page controller, this method is where the chain of page specific events begins. On this page’s geometry changed event, start this page’s specific methods to dynamically populate this page’s content. UXML Files

If you are new to UIToolkit, how it works is Unity has something called a UIBuilder that lets you build these XML documents and define your custom UI or whatever. These UIDocuments can be used as templates, so you can create a UI page and nest the xml document within another’s hierarchy. This is what I did here, each element here is another UI document that is added to the root UI Document as an element of it.

Notebook_Icon.uxml

Corresponding USS page : Notebook_Icon.uss

element description
icon the icon game object that opens the notebook and has the base UIDocument set; the root of the notebook.
closedNotebookMenu uxml template that holds the closedNotebook elements
openNotebookSettings uxml template that holds the openNotebookSettings base elements
openNotebookConversations uxml template that holds the openNotebookConversations base elements
openNotebookInventory uxml template that holds the openNotebookInventory elements
openNotebookTasks uxml template that holds the openNotebookTasks elements

closedNotebookMenu.uxml

Corresponding USS page : closedNotebook.uss

element description
container See container element for openNotebookSettings page
aspect-ratio-closed see aspect-ratio element for openNotebookSettings page
settings a static visual element with a sprite background that is the closed notebook menu’s button to direct the user to the settings page
conversations a static visual element with a sprite background that is the closed notebook menu’s button to direct the user to the conversations page
inventory a static visual element with a sprite background that is the closed notebook menu’s button to direct the user to the inventory page
tasks a static visual element with a sprite background that is the closed notebook menu’s button to direct the user to the tasks page
miskwaa a static visual element with a sprite background that is just an image of Miskwaa
exit a static visual element with a sprite background of an X that is on the closed notebook menu’s top right corner that closes the menu when clicked

openNotebookSettings.uxml

Corresponding USS page : openNotebookSettings.uss

element description
settings-container The container element for each page should have the background image set to the blank openNotebook sprite
aspect-ratio-settings The aspect-ratio element is the absolute positioned container where all of the dynamic elements that get added via C# scripts should be added to as the first level.
settings-title each openNotebook page has a title that is in the uxml document. These are just sprites of the page’s name in a curved styled font.

Each of the 4 notebook pages have the exact same structure so these three same elements will be found in the uxml document for each page and the descriptions here apply to them as well
